- 需求概述
  这是一个 web3 的项目，它的目标是构建一个让非 web3 用户也能体验 web3 魅力的短句接龙的 web 游戏

- 需求功能点
  - 项目在 base 测试网运行
  - 使用 webauth 技术，用户可以无感创建账号
  - 无感为用户创建智能合约钱包，并帮助用户支付 gas
  - 用户需要从一堆短句中选择一个来进行接龙
  - 在前端页面里展示当前的故事，并在故事内高亮用户选择的所有短句
  - 引导用户去测试网水龙头领 eth 充值到公共账户(代付 gas 的账户)

- 技术方案
  - 前端、后端
    - 架构
      - 前端：帮用户创建 EOA 账号，并使用 wagmi 读合约内容比如当前的故事
      - 后端：负责处理前端发送的请求，比如用户选择了一个短句，后端负责接受交易，帮用户创建智能合约账户并支付交易的 gas
    - 选型
      - 使用 NextJs 开发前端和后端应用，并部署在 Vercel 上
      - 使用 webauth 帮用户创建 EOA 账号，为兼容以太坊需要制定算法为 secp256k1
      - 后端使用 zerodev 来帮助用户创建智能合约账户
      - 前端组件库 material-ui
      - 依赖安装全部使用 pnpm
  - 合约
    - 框架与语言
      - 使用 solidity 开发合约
      - 开发与测试框架使用 foundry，执行 foundry 命令前先执行 source /Users/bytedance/.bashrc 来加载环境变量
    - 合约设计
      - 用户智能合约钱包使用 zerodev-sdk 来创建，无需单独设计智能合约钱包合约
      - 只需要开发一个故事接龙的合约，合约里只存储故事短句的index和短句对应的用户address（用于高亮）

- 前端 UI
  - 首页
    - 输入框：位置页面中间靠下，用户不可以直接输入需要在短句列表里选择，选择后回填到输入框
    - 实时故事：页面中间靠上可以滚动
    - 短句列表：入口在输入框内，按钮点击后弹窗里展示展示所有可用的短句，用户需要从这里选择一个来接龙
    - 提交按钮：输入框内，用户点击后提交选择的短句
    - 充值按钮：用户点击后跳转到水龙头页面，引导用户去测试网水龙头领 eth 充值到公共账户(代付gas的账户)
